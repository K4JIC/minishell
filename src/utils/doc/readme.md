# 利用方法
## 関数の利用
```
make all
```
静的ライブラリを作成して、ヘッダファイル(src/utils/includes)と共に使うことで、他ファイルのテストに利用可能。

## 各機能のテスト
```
make ディレクトリ名
```
そのディレクトリ内の関数をテストできる。

例：
```
make msmalloc
```

# 実装内容
## msmalloc
いわゆるガベージコレクタ。既存のmalloc, freeと置き換えて使う。
確保されたヒープ領域をすべて記憶し、mallocが失敗するとすべて解放しプロセスを終わらせる。

メリット
```
ptr = malloc();
if (!ptr)
	return (NULL);
```
のヌルガードが不要になり、たとえば以下のような恩恵が得られる。
- 返り値がchar *の関数を使いやすくなる。
- Normの25行制限を回避しやすい。

デメリット
- リストで実装しているためメモリ消費が多い。（malloc一回につき16byte多い）
- libft内部のmallocもすべてこれで置き換える必要があり面倒。
- malloc失敗時の処理がfree -> exit に固定される。（subjectで許されているか要確認）


## intrusive_list
侵入型リスト（英: intrusive list）。
リスト構造体のひとつ。値を格納するオブジェクトがノードを管理する構造体をもっている。

minishellでリスト構造体を使うにあたって、
用途に応じて毎回新たなリスト構造体を定義するとコードを再利用できない。
かといって、t_list構造体のvoid *contentに値を持たせる形にすると、再利用は可能なものの、ノード追加のたびにリストとcontentで二回のmallocが必要になる。

intrusive_listを使うと上記を解決可能。
言葉で説明するのがむずかしかったので、詳しいことはコード（src/utils/intrusive_list.c）と「侵入型リスト.pdf」をみてください。

## quote
クオートされた文字列の長さを測る関数"get_quote_len()"を実装。
多くの場面で出番が想定されるためutilsに格納した。

- readlineで受け取った文字列のトーカナイズ
- トーカナイズ後のクオートの展開
- 環境変数のvalueのクオート処理（export ABC="abc=" のときなど）
